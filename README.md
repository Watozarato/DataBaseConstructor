# DataBaseConstructor
Конструктор базы данных на бинарных данных, ест много, пользы мало

# Использование
## Создание и настройка БД
Импортируйте объект DB для работы.
DB.create - начать создание базы данных.
* В первом аргументе - максимальное число записей (выше этого предела невозможно)
* Второй аргумент - объект с настройками. Пока что единственное что можно настроить - allocatedRecords - число выделенных записей | начальный предел записей (на такое количество будет выделено памяти)
После этого можете создавать поля для данных в БД.
Существуют 3 основных типа данных для БД - number (с подтипами подобными у TypedArray), string и boolean
Также поля разделены на: Поля уникальных значений, а также поля любых значений (но того же типа).
Поля уникальных значений обязательны у заполнению.
Имя для поля обязательно должно быть уникальным (дубликатов в БД не должно быть)
Типы для полей Number: "Int8" | "Int16" | "Int32" | "Uint8" | "Uint16" | "Uint32" | "BigInt64" | "BigUint64" | "Float16" | "Float32"
Однако учтите, что если у вас V8 не поддерживает например float16, то следовательно применять данный тип нельзя будет.
Типы для полей String: "UTF-8" | "UTF-16"
Даже если в вашей среде нет TextDecoder - движок сможет сохранять данные по стандарту UTF-8.
### Методы добавления полей:
* addFieldUniqueNumberValues(name, type) - добавить поле уникальных Number значений.
* addFieldAnyNumberValues(name, type, defaultValue) - добавить поле любых Number значений. Принимает необязательный параметр defaultValue
* addFieldUniqueStringValues(name, type, maxChars) - добавить поле уникальных String значений. Обязательно указывать maxChars - максимальная длина строки в байтах
* addFieldAnyStringValues(name, type, maxChars, defaultValue) - Добавить поле любых String значений. Обязательно указывать maxChars - максимальная длина строки в байтах. Принимает необязательный параметр defaultValue
* addFieldBooleanValues(name, defaultValue) - добавить поле bool-значений. Не может быть полем уникальных значений.
### Установка колбека на выделение памяти
* setCallBackForAllocateMemory(function) - Принимает функцию, с предустановленным this на объект БД после ее создания. В аргументы функции будут переданы также данные: количество записей на момент вызова этой функции, и максимум записей (указывали в DB.create первым аргументом). Переданная функция будет вызываться в момент когда число записей стало равно числу выделенных записей
### Конец создания БД
* endCreation - Закончить настройку полей, перейти к работе с данными БД
## Работа с данными
### Создание новой записи
* addRecord() - создать запись, аргументы - данных в соотствествии с типами полей в порядке том, как вы определили поля.
### Поиск индекса записи
* findRecordByObject(index, object) - найти индекс записи с данными соотвествующими данным переданного объекта (имя поля объекта - имя поля в БД), начать поиск с index.
* findAllRecordsByObject(index, object) - вернет массив с индексами записей данные которых соотвествуют данным переданного объекта (имя поля объекта - имя поля в БД), начать поиск с index
Есть и другие методы в роде:
* findRecordWithValues
* findRecordByArrayArgs
Но их расписывать не буду, так как вряд ли будете пользоваться ими, ибо в них данные для поиска надо передавать в порядке, как вы задавали поля, что весьма муторно, но может быть полезно, если у вас полей мало.
### Получение данных
* getInfo() - Вернуть объект данных с информацией по БД (ее поля, сколько памяти уже занимает и тд). Нужен для создания БД при использовании метода DB.createFromJSON
* getRecordsInfo() - Получить массив объектов в каждом из которых будут данные записи. Получает ВСЕ записи БД
* getInfoSomeRecords(indexToStart, countRecords) - Получить массив объектов в каждом из которых будут данные записи. Начать с indexToStart и получить ровно countRecords
* getRecordsCount() - Получить число записей в БД
* getFields() - Получить массив записей (это копия, изменение объектов полученных не повлияет на БД)
* getDataFromRecord() - Получить объект с данными записи (ключ - имя записи)
### Изменение данных записи
* setDataOfRecord(indexRecord, ...args) - Изменить данные записи (по индексу) данными из аргументов (работает подобно findRecordWithValues)
### Абстракция
* createObjectAbstractionWithKeyIsUniqueField(fieldName) - создать object-абстракцию над БД, где ключами будут значения из поля уникальных значений (выбирается по его имени).
У этой абстракции 3 метода:
* get(key) - получить объект записи по ключу, обращаясь к ключам (имена полей) которого вы можете менять данные этой самой записи. Запрещено ключ с именем поля уникальных значений на основе которой создана абстракция.
* has(key) - есть ли запись с таким значением в поле уникальных значений (на основе которой создана абстракция)
* createByObject(key, object) - создать новую запись с данными из object (ключ - имя поля), key аргумент - данные для поля уникальных значений. В этом методе также обязательно прописывать данные для полей уникальных значений.
# Примеры использования
* Примитивный пример использования
```js
var db=DB.create(200_000, {
		allocatedRecords:1
	})
	.addFieldUniqueNumberValues("id", "BigInt64", 0n)
	.addFieldAnyNumberValues("freeMoney", "Uint16", 0)
	.addFieldAnyStringValues("username", "UTF-8", 64, "a")
  .setCallBackForAllocateMemory(
		function(currentRecords, maxRecords){
			this.allocateMemoryForRecords(5);
		}
	)
	.endCreation();
db.addRecord(1n);
console.log(db.getRecordsInfo());
```
* Использование object-абстракции
```js
//... Код подобный тому, что выше
var objectAbstraction=db.createObjectAbstractionWithKeyIsUniqueField("id");
objectAbstraction.get(555n).freeMoney=9999;
objectAbstraction.get(105n).freeMoney=550;
objectAbstraction.createByObject(999n, {freeMoney:9999, suka:9});
```
